# Personas

> Review lenses for tailoring, UI clarity, and recruiter verification.

These personas are meant to guide changes to Resume Matcher without inventing candidate experience or over-optimizing at the expense of truthfulness.

## Lineup

| Persona | Primary Question |
|---------|------------------|
| [Job Seeker](job-seeker.md) | Does this feel like a believable transfer into the target role? |
| [UX/Product](ux-product.md) | Can a user understand, edit, and trust the flow? |
| [Hiring Manager](hiring-manager.md) | Would I shortlist this candidate based on this resume? |
| [Recruiter Verifier](recruiter-verifier.md) | Will this survive a recruiter scan and ATS read? |

## How to Use

Run the personas in order when reviewing a new resume-related change:

1. Job Seeker
2. UX/Product
3. Hiring Manager
4. Recruiter Verifier

The last pass should always be the recruiter verifier, because it is the best guardrail against awkward wording, missing links, or formatting that looks polished but screens poorly.
